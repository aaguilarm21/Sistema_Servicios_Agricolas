PROYECTO: Sistema de Servicios Agrícolas

Descripción general:
Sistema web desarrollado en Django para gestionar servicios agrícolas, catálogos de proveedores, empleados, maquinaria, bodegas, artículos y registros operativos. El proyecto incluye autenticación de usuarios, roles de administrador y usuario, formularios de registro operativo y APIs REST básicas para catálogos.

Estructura del proyecto:
- `miproyecto/`: configuración principal de Django, URLs globales y vistas principales.
- `accounts/`: aplicación local que contiene modelos, formularios, vistas, URLs y APIs para usuarios, catálogos y autenticación.
- `Backend/`: vistas de backend que renderizan páginas, controlan acceso y procesan formularios operativos.
- `Frontend/`: plantillas HTML y activos estáticos (CSS, JS, imágenes).
- `BaseDatos/`: base de datos SQLite (`db.sqlite3`) usada por la aplicación.
- `.txt/`: documentación interna como `ER_Proyecto.txt`, `ER_Diagramas.txt` y `proyecto_descripcion.txt`.

Configuración relevante:
- `miproyecto/settings.py` define la base de datos SQLite en `BaseDatos/db.sqlite3`.
- Las plantillas se cargan desde `Frontend/templates` y los archivos estáticos desde `Frontend/static`.
- Se usa el sistema de autenticación de Django con `django.contrib.auth` y vistas personalizadas para el login.

URLs y funcionalidad principal:
- `/`: página de inicio (`home.html`). Requiere inicio de sesión.
- `/accounts/login/`: login personalizado con `CustomLoginView` y `CustomAuthenticationForm`.
- `/accounts/signup/`: registro de usuario con formulario de creación de cuentas.
- `/accounts/ajax-login/`: login vía AJAX.
- `/accounts/api/.../`: APIs para listar, crear, actualizar y eliminar proveedores, empleados, usuarios, maquinaria, bodegas, artículos y catálogos auxiliares.
- `/modulos/`: panel principal de módulos. Muestra opciones según rol de usuario.
- `/configuracion/`: acceso a la configuración de catálogos. Solo administradores.
- `/registros/`: formulario de ingreso de registros operativos. Usuarios y administradores.
- `/registros-data/`: listado de registros operativos. Solo administradores.
- `/registros/<int:registro_id>/editar/`: editar un registro operativo existente. Solo administradores.
- `/registros/<int:registro_id>/borrar/`: eliminar un registro operativo. Solo administradores.
- `/usuarios/`: módulo de gestión de usuarios. Solo administradores.
- `/usuarios/crear/`: endpoint JSON para crear usuarios con perfil y rol. Solo administradores.
- `/usuarios/creados/`: listado de usuarios creados.
- `/operacion/`: módulo de operación agrícola. Accesible para usuarios autorizados.
- `/reportes/`: módulo de reportes. Solo administradores.

Autenticación y roles:
- Usa el modelo `User` estándar de Django.
- `CustomAuthenticationForm` personaliza mensajes de error y muestra advertencias en español.
- `ajax_login` realiza autenticación por JSON y retorna errores específicos.
- Los usuarios pueden pertenecer a grupos de Django como `Admin` o `Usuario`.
- Las funciones `user_is_admin()` y `user_is_user_or_admin()` validan el acceso a cada vista.
- Un usuario inactivo recibe el mensaje: `USUARIO INACTIVO, COMUNIQUESE CON EL ADMIN, PARA ACTIVARLO`.

Modelos principales (`accounts/models.py`):
- `UserProfile`: datos extendidos de usuario (código, puesto, timestamps).
- `Proveedor`: datos del proveedor, NIT, razón social, régimen tributario y crédito.
- `Empleado`: empleados con empresa, nombre, CUI, puesto y nombre de puesto.
- `Maquinaria`: maquinaria registrada con código, combustible, proveedor, tipo, marca, serie y placa.
- `Bodega`: nombre, unidad de medida, capacidad y observaciones.
- `Articulo`: código, descripción, unidad, categoría y stock.
- Catálogos auxiliares: `Labor`, `Cuenta`, `UnidadMedida`, `Variedad`, `TipoMaquina`, `Marca`, `Municipio`, `Auxiliar`.
- `RegistroOperativo`: registro de boletas operativas con datos de servicio, proveedor, máquina, finca, lote, costos, personal y observaciones.

Lógica de negocio en `Backend/views.py`:
- `home()`: renderiza la página principal.
- `modulos()`: muestra el panel de módulos y pasa `is_admin` a la plantilla.
- `configuracion_catalogos()`: protege el acceso para administradores.
- `registros_operativos()`: gestiona GET y POST del formulario operativo; crea `RegistroOperativo` con campos opcionales.
- `registros_operativos_data()`: lista todos los registros para administradores.
- `editar_registro_operativo()`: actualiza campos de un registro operativo existente.
- `borrar_registro_operativo()`: elimina un registro operativo.
- `usuarios()`: carga la vista de usuarios para administradores.
- `crear_usuario()`: endpoint JSON que crea un usuario, asigna grupo, crea perfil y valida datos.

Autenticación y APIs en `accounts/views.py`:
- `CustomLoginView`: usa `CustomAuthenticationForm` y la plantilla `registration/login.html`.
- `ajax_login()`: valida credenciales por AJAX y maneja usuarios inactivos.
- `signup()`: crea usuarios desde un formulario, genera username basado en nombre completo y asigna perfil.
- `buscar_usuario_por_codigo()`: busca empleados por código para autocompletar datos.
- APIs REST para catálogos: proveedores, empleados, usuarios, maquinaria, bodegas, artículos, labores, cuentas, unidades, variedades, tipos de máquina, marcas, municipios y auxiliares.

Formularios y validación:
- `CustomAuthenticationForm`: muestra mensajes de error en español.
- `AdminUserCreationForm`: valida nombres completos, genera `username` automáticamente, verifica contraseñas y permite seleccionar rol.

Frontend y plantillas:
- `Frontend/templates/home.html`: inicio del sistema después del login.
- `Frontend/templates/modulos.html`: panel de módulos con control de acceso y botones de sesión.
- `Frontend/templates/configuracion_catalogos.html`: página de administración de catálogos.
- `Frontend/templates/registros_operativos.html`: formulario para crear boletas operativas.
- `Frontend/templates/registros_operativos_data.html`: tabla de registros guardados.
- `Frontend/templates/usuarios.html`: vista de administración de usuarios.
- `Frontend/templates/registration/login.html`: formulario de login.
- `Frontend/templates/registration/signup.html`: formulario de registro.

Estilos y scripts:
- `Frontend/static/css/modulos.css`: estilos del panel de módulos, modal y barra de acción.
- `Frontend/static/css/registros.css`: estilos de formularios y listados de registros.
- `Frontend/static/js/common.js`, `notificaciones.js`, `usuarios.js`, `usuarios_creados.js`: scripts para AJAX, notificaciones y gestión de usuarios.

Base de datos y persistencia:
- La aplicación usa SQLite y el archivo `BaseDatos/db.sqlite3`.
- Hay migraciones existentes en `accounts/migrations/`.
- Los modelos se guardan mediante las vistas de Backend y las APIs REST.

Notas importantes:
- El proyecto está configurado para desarrollo (`DEBUG = True`). No es seguro para producción sin ajustes adicionales.
- El sistema utiliza datos de catálogos auxiliares para autocompletar fincas, labores, variedades y municipios en los registros operativos.
- El registro operativo en la vista guarda referencias por texto, no implementa claves foráneas estrictas en todos los campos.
- La gestión de usuarios permite activar, desactivar y eliminar usuarios desde la API propia.

Resumen funcional:
1. El usuario inicia sesión en `/accounts/login/`.
2. Accede al menú de módulos en `/modulos/`.
3. Los administradores pueden gestionar catálogos, ver registros y administrar usuarios.
4. Cualquier usuario autorizado puede crear boletas en `/registros/`.
5. El sistema dispone de APIs internas para consulta y mantenimiento de catálogos agrícolas.
